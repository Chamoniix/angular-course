CHAPITRE 6

Services

Et le backend ?

Qu'est ce qu'un service ?

  • Centralise du code pour un module
  • Partage des informations globales
  • Utilisé dans plusieurs parties de l'application

Exemple: Students, Authentification, création d'utilisateurs, déconnexion, etc.

Utilisation

Doit être injecté

Niveau d'injection ∝ nombre d'instance

  • app.module : Tous les composants & services
  • app.component : Tous les composants
  • my-component : Composant et ses enfants

Nombre d'instances

2 premiers cas : singleton

Dernier cas : 1 instance composant = 1 instance service.

Utilisation

Injection de dépendance

providers: [StudentService]
constructor(private studentService: StudentService) {}

Utilité

  • Pas de duplication
  • Partage des données entre les composants
  • Exécute des tâches
  • Récupère les données

Génération du service

ng generate service students

Remarque : On ne suffixe jamais pas '-service'
